Hello Hexo, Hello My Blog

发表于 2017-02-10

历尽千辛万苦还是把你给配置好了。你好，我的Blog。

还是选择记录一下基于Windows的配置的过程。

##

创建Github域名和空间

Github账号的username会影响到你的域名，然而在创建完成后同样也是可以去修改你的username。

账号创建好后，需要创建一个仓库(repository)来储存我们的网站，创建仓库时，仓库名（repository name）username.github.io 的username 一定与前面的你账号的 username 一致，记住你的 username 下面会用到。

Hexo简介

Hexo 可以说是目前最流行的博客框架了，基于Nodejs，更多信息可以google，下面需要安装的工具包括 Git，Nodejs，Hexo。

安装Git

这个windows就十分简单了，直接下载安装即可。我是下载的这个版本的.
然后就是不停的下一步就可以。

安装Node.js

  1. Node.js的下载与安装

    node.js的下载地址。自己选择路径，安装完以后，进入到根目录下。

    打开cmd，

    G:/> npm config set prefix “G:\Program Files\Programe\nodejs\node_global”

    G:/> npm config set cache “G:\Program Files\Programe\nodejs\node_cache”
    ```

    可以通过输入G:/> npm config get prefix 来查看设置是否成功。

  2. 设置系统环境变量

    由于前面改变了module的默认地址，所有用户变量都需要跟着改变一下(用户变量PATH修改为刚才我们设置的全局模块存放路径(即G:\Program Files\Programe\nodejs\node_global),否则在使用module的时候会再cmd中出现XXX不是内部或外部命令，也不是可运行的程序或批处理文件 错误。

    用户变量设置

    变量名:Path

    变量值:xxxx;G:\Program Files\Programe\nodejs\node_modules\npm;G:\Program Files\Programe\nodejs\node_global

    注:xxxx为你之前的用户变量，只需要在后面加上即可。

    同时在系统变量添加NODE_PATH变量，使用全局安装时，首先会在这个目录去寻找模块。

    变量名： NODE_PATH

    变量值： G:\Program Files\Programe\nodejs\node_modules

    环境变量配置完成了，记得去重启一下电脑,偷懒可是会增加麻烦的

  3. 测试

    在cmd中输入G:/> npm install bower -g，待安装成功后，输入bower -v， 出现版本号即为配置成功。

  4. 没了

    开始在安装的时候都挺顺利的，可是在修改完module的路径后，然后去修改环境变量的时候遇到问题了。如果你也遇到相同的问题了，记得在从头看一下这部分。

安装 Hexo

一句命令G:/> npm install hexo-cli -g

至此必须的工具就已经安装完成了，然后就是生成博客，上传至Github上的repository中了。

##

生成、编写、发布

以下操作都是在cmd中进行

  1. 生成博客

    博客的初始化，这时候就用到了上面注册github时的username。在这之前，首先将自己当前的cmd路径切换到你想将你的blog项目存储的位置上，这个操作就是为了把hexo初始化到你本机当前路径下。当然现在这文件名也就是username.github.io ,你也可以换成其他的

    hexo init username.github.io

  2. 更改配置

    hexo在完成后，你就可以去更换各式各样的主题了。先进入到刚生成的Hexo目录下

    G:/> cd username.github.io
    G:/> git clone https://github.com/iissnan/hexo-theme-next themes/next

    这是一个极简的主题，还有很多主体在这里，想用哪个就用哪个

    基础配置： 在你的目录下，开始编辑_config.yml,下面的需要去修改。配置的键值之间一定要有空格，简单的说就是冒号后面空一格。Hexo配置说明


    title: Chrysalis2H’s Blog        //你博客的名字author: chrysalis2h                //你的名字
language: zh-Hans                //语言 中文
theme: next                       //刚刚安装的主题名称
deploy:
  type: git                    //使用Git 发布
  repo: https://github.com/username/username.github.io.git        // 刚创建的Github仓库
</pre>
** 主题配置： **    配置文件在Hexo根目录下的/themes/next/_config.yml`。next 配置说明

  3. 准备工作基本已完成

    你的blog文章目录在根目录下的/source/_posts/里，里面的hello-world.md已经等你好久了。文章用Markdown来编写，也不用再去写一篇了，直接来准备测试一下看看吧。

  4. 测试及发布

    G:/> hexo s

    启动成功后，就可以在浏览器中打开了，地址http://localhost:4000。
    终于看到了T_T

    接下来就准备要发布到你开始注册的username.github.io的仓库中了(这个终于派上用场了-_-), 安装hexo-deployer-git自动发布工具

`G:> npm install hexo-deployer-git --save`

终于到了发布的时候了，

`G:/> hexo clean && hexo g && hexo d`

要是这是第一次使用，部署会不成功，终端中会让你 `please tell me who you are`，此时你只需要输入

`G:/> git config --global user.email "github的邮箱"`
`G:/> git config --global user.name "github的username"`

然后再执行发布命令就行了。每次写完blog，执行以下命令就发布上去了，so easy。

这时候你就可以去`https://username.github.io`,看看你的blog了。

我遇到的问题

不知道经过上面那些你打开没，反正我第一次是没有成功的打开blog，项目也传上去了，测试也成功了，然后就是没打开。后来就又从头来了一遍，将仓库名和我的username命名为一致(这个真的很重要)。其实用hexo+github来搭建一个blog还是很方便的
